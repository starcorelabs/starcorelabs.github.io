Technical Writing

2019-05-04

Throughout my IT career I’ve written all sorts of technical documents including: Project Specifications, System Requirements, SOP’s, Style Guides, Content Guidelines, and a UX Developer Start Guide to help new employees get up to speed quickly.

Technical Writing is desperately needed in web development. I’ve wasted hours of time searching through poorly written style guides and project specifications looking for answers to simple design and coding questions. In our information technology driven society, documentation should never be a bottleneck to productivity.

I write because I enjoy it and find it useful. It helps me keep track of things. Especially when I’m working on two or three projects at the same time. Referencing notes allowed me to jump back in where I left off. Colleagues made use of the documentation I wrote. Which made me happy.
