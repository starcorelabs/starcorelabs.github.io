Chrome OS

2018-12-01

Chrome OS is a minimal operating system developed by Google in 2009. It uses a modified linux kernel and the Google Chrome web browser as its main user interface.

It requires a Google account and an internet connection to login to Chrome OS. Relying on Google’s Cloud infrastructure for data storage and web applications for productivity. Some apps can be used offline. Because most Chrome OS devices use low-powered hardware with limited internal storage the system is most useful when connected to the internet.

Back when it was first announced I was intrigued by the idea of a Google owned Operating System. I envisioned a robust version of Linux similar to Ubuntu with a streamlined look.

A few years ago when I tested a colleague’s Chrome OS laptop I was majorly disappointed.

It was a weak locked down version of linux with no root privileges to make changes or access to the internal file system. After 20 minutes of testing, it became clear, without an internet connection this laptop would be useless to me.

I’m a power user and want full control over my operating system. I enjoy working on and storing my files locally.

In my area the internet is mostly stable but there are outages. On my Linux systems this is not a problem as I have access to my local files and can continue working until my connection is restored. Relying on cloud-based storage and web apps is not a viable option for me.

Furthermore, I don’t want to be tied to Google’s ecosystem and tracking. As I become more aware of issues with online security and privacy. It only seems logical to take responsibility for securing my own data instead of relying on the benevolence of a for profit multi-billion dollar multinational corporation.

As a front-end developer my workflow requires me to use applications that either do not work on Chrome OS or have no web app alternative.

The use case for Chrome OS is for light web browsing, email, YouTube, social media, and online document creation. Anything more complex is too much for this limited operating system and hardware combination.

Chrome OS came out as an alternative to the rise of Netbooks. However, by the time a working version of Chrome OS was released the Netbook trend had ended. Smart phones had matured and become the main data consumption device used by the majority of people.

In my opinion for Chrome OS to survive and grow it will need to adapt to the needs of early adopters and power users. Because there is no mass market appeal for an Operating System that is not as powerful or useful as a standard smart phone.
