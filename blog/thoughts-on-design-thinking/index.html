Thoughts on Design Thinking

2023-03-04

TL;DR

Design Thinking is a process for solving design-related issues. It is not a general purpose problem solving method or a replacement for a project management framework.

What Is Design Thinking?

Design Thinking is a user focused design-related problem solving process that can be learned in less than five minutes.

It is very similar to Techincal Support Troubleshooting which can be broken into four steps.

  1. Define the problem - Who is this affecting? Are we solving the right problem?
  2. Gather more details - Root cause analysis. Eliminate variables.
  3. Reproduce the problem, if possible - Testing.
  4. Attempt a fix based on the evidence.

Generally speaking, this process generates a 90% success rate for fixing technical problems.

Design Thinking adds an extra step to generate solutions for design-related issues. There are five steps in the Design Thinking Process:

  1. Emapthize
  2. Define
  3. Ideate
  4. Prototype
  5. Test

Its fascinating how Universities that teach UX/UI Design call these steps "The UX Design Process". Different words, same process.

Some organzations have renamed the five steps:

  1. Product Definition
  2. Research
  3. Analysis
  4. Design
  5. Validation

Again, different labels, same steps.

The question is why change the names? The answer is always money.

How can a company charge people for an over-engineered Design Thinking course if they can learn it by watching a free five-minute YouTube video? Answer, they can't.

Unless, they rename everything to sound more complicated than it is.
Same process, different words.

Choose The Right Tool For The Job

I like Design Thinking because its good for solving complex design issues. But, like any tool, its not meant to be used for everything.

Its important to choose the right tool for the job. If you have a hammer it doesn't make all your problems nails.

Some individuals are looking for a "One Size Fits All" process that absolves them of thinking, using common sense, or accountability. Hoping that by using the process it will give them the answers and direction they need. Which of course it cannot.

Being creative is hard and not everyone can do it. UX/UI Design requires education in rational decision-making, problem solving skills, understanding user psychology, design principles, visual creativity, and the technical understanding of how web applications function.

Design Thinking, while useful, is one of many tools in the UX/UI Designer's Toolbox.
