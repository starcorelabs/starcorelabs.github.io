## Behavior Questions
### 1. Tell me about your work experience. For the last 6 years I worked as a professional UI/UX Developer in team environments collaborating on large corporate web applications using Scrum / Agile methods.

I’ve designed wireframes, coded views in HTML5, CSS3, and Bootstrap 4. While using JavaScript and jQuery for functionality and validation.

I’ve also written documentation and contributed to style guides.

### 2. Where did you learn your skills? I learned from books, online video courses, and on the job from other developers in team environments. I'm always learning new things.
### 3. How do you keep updated on changes in the industry? I use [Feedly](https://feedly.com/) and follow a collection of technology related RSS feeds. I use Twitter and YouTube to follow developers who teach what they know.
### 4. Are you currently working? I am not working at the moment. I haven't had a job since my last contract ended. I really want a job and would be grateful if you hire me.
### 5. Why did you leave your last job? It was a short-term contract that ended. I wanted to be hired on full-time but it was only a short-term contract. I learned a lot and liked everyone I worked with.
### 6. What did you do in your last role? I was part of a three person UX Development team tasked with fixing code errors in front-end views for a Student Loan Repayment web application.

We compared the pages that were built by the offshore team against prototypes to find errors.

We rewrote Bootstrap, HTML, and SCSS code to correct the placement and spacing of components on pages. We tested our code, visually and with software, to make sure the components resized correctly on desktop, tablet, and mobile devices.

After 5 months of fixing bugs the executive team moved creation of front-end views from offshore to our department. This reduced the time it took for design, coding, testing, bug fixing, and deployment.

### 7. Why do you have a gap in your employment? I started applying to UX Development positions immediately after my last contract ended. Most positions required a Bachelor's Degree and 3+ years experience in React, Angular, and/or Vue. I was passed over because I lacked a degree or experience with those JavaScript frameworks.

But, I’m learning and improving my skills in those areas.

### 21. Tell me about a time you tried to do something and failed. While working at HCCMIS I was part of the team assigned to redesign the company's main website. I was the only person writing CSS and Bootstrap code for the project. My manager Bill was focused on writing the PHP code for the back-end Cascade CMS the website would run on.

It was a large corporate project. The CSS was becoming difficult to keep organized. I asked Bill if I could create a SASS version locally on my work computer. That way I could take advantage of variables, mixins, nesting, and organizing the style sheets by category. I assured him it would output cross-browser compatible CSS code. Plus, it would allow me to make site wide changes when stake holders decided to change something without spending a day hunting through code.

He said, “No. I might have to make changes to the CSS and I don’t have time to learn SASS.”

I asked him if he’d give me the chance, on my own time, to create a SASS version of the site to show him the benefits. He agreed. Over a weekend I spent a full Saturday and most of Sunday converting the CSS to SASS and cleaning up the code base in the process. Reducing complexity, removing redundancy, and fixing errors.

During our next Monday morning meeting I showed him how Gulp running on Node-SASS outputs a standard CSS file. I explained all the benefits and how using SASS helped me track down a few bugs and fix them. After listening for a few minutes he said, “No. We are going to keep using standard CSS.” Then he walked away. No explanation, rationale, or reasons.

Keep in mind, through out the whole project he never wrote a single line of CSS code.

New HTML5-specific tags will not be interpreted by the browser.

3. What is the difference between span and div?

The difference is that span gives the output with display: inline and div gives the output with display: block.

4. What’s the difference between the svg and canvas elements?

The svg element is a container for SVG graphics. The canvas element allows you to draw 2D graphics on the fly using JavaScript.

External style sheets are useful to organize styles that can be reused in multiple places.

3. Explain the term Responsive web design.

It is a method in which we design and develop a web page according to the user activities and conditions which are based on various components like the size of the screen, portability of the web page on different device etc.

My Questions

### Tech Questions - Who are the stake holders? - Will I have one manager or many? - Do you have dedicated project managers? - What criteria is used to decide priority of tasks? - Do you use project management software like [Trello](https://trello.com/en-US)? - Do you use a ticketing system to assign tasks? - Do you follow industry best practices? - How do you organize your CSS/SCSS code base? - Do you use git for revisioning and code deployment? - Does each developer maintain their own change log or is it automated? - Who writes the design briefs, style guides, and requirement documents? - If the end results are the same, can I use [Linux Mint](https://linuxmint.com/) as my OS?
### General Questions - How did you come to work for this company? - What do you love most about working for this company? - Who is the most successful recent hire and why are they successful? - Who didn't succeed as a new hire and why? - What is the company's biggest challenge this year and how will this role help overcome it? - How will I measure my performance to know I'm having a positive impact on this challenge? - What additional skills do you wish I had that would make me a better fit for this role? - What are the next steps?
